# LazyQMK-aopx.11 Fix Report

## Bug
`generate_combo_code` in `src/firmware/generator.rs` (line 1283) used `.expect("Base layer must exist for combos")` on `self.layout.get_layer(0)`. A layout without layer 0 (user input edge case) would panic with a Rust backtrace instead of returning a clean error.

## Fix Applied

`src/firmware/generator.rs` — `generate_combo_code()` at line ~1280:

```diff
-            let base_layer = self
-                .layout
-                .get_layer(0)
-                .expect("Base layer must exist for combos");
+            let base_layer = self.layout.get_layer(0).ok_or_else(|| {
+                anyhow::anyhow!(
+                    "Cannot generate firmware: layout '{}' has no base layer (layer 0). \
+                     A keyboard layout must always have a base layer.",
+                    self.layout.metadata.name
+                )
+            })?;
```

`anyhow::{Context, Result}` is already imported (line 11), so no additional `use` is required.
